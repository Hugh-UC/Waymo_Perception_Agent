# AI Usage Disclosure

## Summary
This project utilized generative AI (Google Gemini) strictly as an assistant for documentation formatting, type hinting, and debugging. All core logic, system architecture, UI/UX design, and primary project documentation were authored entirely by the student.

## Detailed Usage

### 1. Documentation & Comments
*   **Docstrings:** Gemini 3.1 Pro was used to format, complete, and refine Python docstrings across the codebase to ensure strict adherence to industry standards (PEP 8 / PEP 257).
*   **Inline Comments:** The AI improved or suggested comments.
*   **Secondary Directory READMEs:** AI was utilized to generate the basic placeholder `README.md`files found within specific subdirectories (e.g. `data/`). These serve only to provide a brief, surface-level description of the directory's contents.
*   **Primary Documentation Exemption:** The extensive, main project `README.md` located in the root GitHub directory was written entirely by the author, with no AI generation used for its structural or detailed content.

### 2. Code Quality & Typing
*   **Type Hints**: AI was used to check and strictly complete missing Python type hints throughout the backend code to ensure robust static typing.
*   **Debugging**: The AI was utilized as a troubleshooting tool to analyze error outputs, resolve syntax issues, and suggest fixes for edge cases during the development of the server and front-end logic.

### 3. Verification
The author manually reviewed, tested, and verified all AI-assisted code annotations, type hints, and basic documentation to ensure complete accuracy and alignment with the project's original architectural intent.
